## Variables d'environnement à configurer

Dans le dashboard Vercel, ajouter ces variables :

```bash
# Stripe (obligatoire)
STRIPE_SECRET_KEY=sk_live_..._votre_clé_secrète
STRIPE_WEBHOOK_SECRET=whsec_..._votre_secret_webhook
STRIPE_PUBLISHABLE_KEY=pk_live_..._votre_clé_publique

# Supabase (obligatoire)
SUPABASE_URL=https://votre-projet.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Configuration précommande
PREORDER_GOAL=500
```

Pour le développement local :

## Configuration Supabase

### 1. Créer le projet Supabase
1. Aller sur https://supabase.com
2. Créer un nouveau projet
3. Noter l'URL et la Service Role Key

### 2. Exécuter le SQL
Dans le dashboard Supabase → SQL Editor, exécuter le contenu de `supabase_setup.sql` :
```sql
CREATE TABLE IF NOT EXISTS preorders (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    stripe_session_id TEXT UNIQUE NOT NULL,
    payment_intent_id TEXT,
    email TEXT NOT NULL,
    offer TEXT NOT NULL CHECK (offer IN ('standard', 'tirage-offert', 'edition-signature')),
    amount_total INTEGER NOT NULL,
    currency TEXT NOT NULL DEFAULT 'eur',
    full_name TEXT,
    shipping_address TEXT,
    postal_code TEXT,
    city TEXT,
    paid_status TEXT DEFAULT 'pending' CHECK (paid_status IN ('pending', 'completed', 'failed', 'cancelled')),
    source TEXT DEFAULT 'oradia-precommande',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
-- ... (reste du SQL dans supabase_setup.sql)
```

## Configuration Stripe Dashboard

### 1. Créer les produits
Dans votre dashboard Stripe, créer 3 produits :

1. **Standard** - 38.00€
2. **Tirage Offert** - 44.00€  
3. **Édition Signature** - 52.00€

### 2. Configurer le webhook
- URL : `https://votre-domaine.vercel.app/api/stripe-webhook`
- Événements à écouter : `checkout.session.completed`
- Secret : Copier la valeur et l'ajouter dans Vercel

## Déploiement

### 1. Installer les dépendances
```bash
npm install @supabase/supabase-js stripe
```

### 2. Mettre à jour vercel.json
Remplacer le contenu de `vercel.json` par celui de `vercel.precommandes.json` :
```bash
cp vercel.precommandes.json vercel.json
```

### 3. Déployer
```bash
git add .
git commit -m "feat: migrate preorder system to Supabase"
git push origin main
```

### 4. Configurer les variables Vercel
Dans le dashboard Vercel → Settings → Environment Variables :
- Ajouter toutes les variables requises
- Redéployer après ajout
